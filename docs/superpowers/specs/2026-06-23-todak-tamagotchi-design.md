# 🐻 토닥곰 다마고치 설계 — "토닥곰 키우기"

- **날짜:** 2026-06-23
- **표면:** 웹 (GitHub Pages, `docs/play/`)
- **단계:** 곰 우주 게임 로드맵 2/3 (① 미니게임 ✅ → ② **다마고치** → ③ Phaser)
- **상태:** 설계 승인됨, 구현 계획 대기

## 1. 목적 / 한 줄 요약

토닥곰을 **돌보며 키우는** 캐주얼 다마고치. 밥·놀이·토닥·재우기로 곰을 보살피고,
아기에서 노년까지 성장을 함께한다. "혼내지 않고 토닥이는" 곰 우주 톤을 유지 —
**방치해도 곰은 죽거나 화내지 않는다.** 스탯이 비면 잠깐 시무룩/졸릴 뿐, 돌아오면 다시 행복해진다.

## 2. 핵심 철학 (확정)

| 축 | 결정 |
|---|---|
| 방치/돌봄 | **하이브리드** — 스탯은 떨어지되 곰은 죽지 않음. 비면 시무룩/잠듦 상태로만 진입, 돌보면 회복+행복 |
| 시간 모델 | **오프라인 실시간 진행** — `localStorage`에 마지막 접속 시각 저장, 재방문 시 경과시간만큼 스탯 감소 반영 |
| 톤 | 느긋·부드러움. 하루 1~2번만 들여다봐도 충분. 압박 없이 정 붙이는 동반자 |

## 3. 스탯 4종 (확정)

각 0~100. 채우는 행동(버튼)과 1:1.

| 스탯 | 아이콘 | 채우는 행동 | 비었을 때 신호(말풍선) |
|---|---|---|---|
| 배부름 (hunger) | 🍯 | 밥주기 | 배고파 🍯 |
| 즐거움 (fun) | ⚽ | 놀아주기 | 심심해 ⚽ |
| 마음 (heart) | 💗 | 토닥이기 | 외로워 💗 |
| 기력 (energy) | 😴 | 재우기 | 졸려 😴 |

- **감소 속도(느긋):** 각 스탯 시간당 약 **−6/시간**. 가득(100)에서 ~11시간이면 시무룩 영역(<30), ~16시간이면 0.
- **0이어도 죽지 않음.** 가장 낮은 스탯이 곰 표정·말풍선으로 "나 이게 필요해"를 표현.
- **돌봄 1회:** 해당 스탯 **+35**(상한 100). 직후 **행복 표정 + ❤️ 반응** 1.5초.
- **연타 방지:** 행동별 짧은 쿨다운(예 1.5초) — 한 번에 다 채우지 못하게, 들여다보는 리듬 유지.

## 4. 성장 (확정)

**태어난 후 실제 경과일(age) 기준.** 돌봄과 무관하게 진행(느긋 철학 — 방치해도 성장은 멈추지 않음).

| 단계 | 조건(age) | 스프라이트 접두 |
|---|---|---|
| 아기곰 | < 1일 | `baby` |
| 소년곰 | < 3일 | `child` |
| 어른곰 | < 7일 | `adult` |
| 노년곰 | ≥ 7일 | `elder` |

- 단계 전환 시 **⭐ 축하 연출**(짧은 토스트 + 반짝) 후 스프라이트 교체.
- 약 1주면 전 단계를 맛본다(토이 프로젝트 호흡).

## 5. 표정 상태머신 (확정)

곰 스프라이트는 `${stage}_${pose}.png` 16종 사용. 매 프레임 우선순위로 pose 결정:

1. 방금 돌봄(flash 타이머 > 0, 1.5초) → **happy(행복)**
2. 기력 < 25 → **sleep(졸림)** — "재워줘"
3. 다른 스탯 중 최소값 < 30 → **sad(시무룩)** — "돌봐줘"
4. 그 외 → **content(평온)**. (전체 평균 ≥ 85면 happy로 idle)

→ 제작된 4표정(content/sad/sleep/happy)이 그대로 1:1 매핑. **추가 아트 0.**

## 7. 구현 접근

**순수 HTML + Canvas + requestAnimationFrame (바닐라 JS, 의존성 0, 빌드 없음).**
① 꿀단지게임과 동일 스택·패턴. 핵심 로직을 DOM/캔버스에서 분리해 **노드에서 단위 테스트 가능**하게.

대안 검토 후 제외:
- **DOM/HTML 위젯 방식:** 스탯바·버튼은 DOM이 쉬우나, 곰 방 렌더·말풍선·연출은 캔버스가 일관됨. 캔버스로 통일해 ①과 루프 패턴 재사용.
- **지금 Phaser/프레임워크:** ③단계로 미룸. 다마고치엔 과함(YAGNI).

## 8. 모듈 구조 (관심사 분리)

```
docs/play/
├ tamagotchi.html       ← 셸: 캔버스 + rAF 루프 + localStorage 배선 + 버튼 오버레이
├ tama-core.js          ← 순수 로직(DOM 없음): 스탯 감소·돌봄·성장·표정·직렬화
├ tama-renderer.js      ← 캔버스 그리기: 스탯바·곰(단계+표정)·말풍선·버튼·연출
├ tama-input.js         ← 버튼/탭 → 의도(feed|play|pat|sleep|null)
└ assets/
   ├ todak.js           ← bear-core 복사본(대사·팔레트 재사용)
   └ poses/             ← {baby,child,adult,elder}_{content,sad,sleep,happy}.png (16) + src/(원본·추출기)
```

| 모듈 | 하는 일 | 의존 | 인터페이스 |
|---|---|---|---|
| `tama-core` | 상태 1개 받아 다음 상태 반환. 순수 함수 | 없음 | `createState(opts)`, `applyElapsed(state, ms)`, `tick(state, dt)`, `care(state, type, nowMs)`, `growthStage(state)`, `poseFor(state)`, `serialize/deserialize` |
| `tama-renderer` | 상태 → 캔버스. 곰 단계+표정 교체, 스탯바·말풍선·버튼 | poses, ctx | `draw(ctx, state)` |
| `tama-input` | 버튼/탭 → 의도 | DOM 이벤트 | `attach(el, hitBoxes)`, `read() -> type|null` |
| `tamagotchi.html` | rAF 루프로 배선 + 저장/복원 + 성장 토스트 | 위 셋 + todak.js | — |

## 9. 데이터 흐름

```
앱 시작
   → state = load() ?? createState({ bornAt: now })
   → applyElapsed(state, now - state.lastSeen)   // 오프라인 감소 일괄 반영

requestAnimationFrame(dt)
   → intent = input.read()                  // 'feed'|'play'|'pat'|'sleep'|null
   → if intent: care(state, intent, now)    // 스탯 +35, flash=1.5s, 쿨다운
   → tick(state, dt)                         // 시간당 −6 비례 감소, flash 감소
   → stage = growthStage(state); 단계 변하면 ⭐ 토스트
   → renderer.draw(ctx, state)
   → 주기적으로 save(state)                   // lastSeen 갱신
```

**상태 모양 (단일 객체):**
```js
{
  name,                 // 곰 이름(기본 "토닥곰")
  bornAt,               // 태어난 시각(ms) — 성장 계산 기준
  lastSeen,             // 마지막 저장 시각(ms) — 오프라인 감소용
  stats: { hunger, fun, heart, energy },  // 각 0~100
  flash,                // >0이면 방금 돌봄 → happy 포즈(초)
  cooldown: { feed, play, pat, sleep },   // 행동별 남은 쿨다운(초)
}
```

- **돌봄:** `care(state, type)` → 해당 스탯 +35(클램프), `flash=1.5`, 쿨다운 설정
- **감소:** `tick`에서 각 스탯 `-= 6 * dt/3600s`, 0 클램프
- **오프라인:** `applyElapsed(ms)` = 경과시간을 같은 식으로 일괄 차감(상한: 너무 오래면 0 수렴)
- **성장:** `growthStage` = `now - bornAt`를 일 단위로 환산해 단계 반환

## 10. 상태·저장 / 캐릭터 연동

- **저장:** `localStorage['todak.tama.v1']` = 위 상태 직렬화(JSON). 없으면 새 곰 생성.
- 곰 대사·팔레트는 `assets/todak.js`(bear-core 복사본)에서 가져와 톤 일관성 유지.
- 말풍선 문구는 위로/응원 톤(ko/en). 절대 혼내지 않음.

## 11. 반응형 / 모바일

- 캔버스는 컨테이너에 맞춰 크기 조정, **세로형 비율(3:4)**.
- `devicePixelRatio` 반영해 또렷하게.
- 입력: 버튼 탭/클릭(PC·모바일 공용). 버튼 히트박스는 논리좌표 기준.
- 탭 전환 시 `visibilitychange`로 일시정지(dt 폭주 방지), 복귀 시 경과시간 재적용.

## 12. 에러 처리 (조용히 견디기)

- `localStorage` 막힘(시크릿) → try/catch, 메모리 상태로 폴백(세션 한정).
- **스프라이트 로드 실패** → `todak.js` ASCII 얼굴·색 원으로 폴백 → 그래도 돌봄 가능.
- 첫 프레임/복귀 dt 튐 방지(틱 dt 상한 클램프). 오프라인 경과는 별도 일괄 처리.
- 깨진/구버전 저장값 → 스키마 검증 후 실패 시 새 곰으로 초기화.

## 13. 테스트

- **순수 로직(`tama-core`)**: 노드에서 프레임워크 없이 검증
  - 돌봄 시 해당 스탯 +35, 상한 100 클램프
  - `tick`/`applyElapsed`로 시간 비례 감소, 0 하한 클램프
  - 오프라인 경과 적용 정확성(예: 6시간 → −36)
  - `growthStage` 경계(1·3·7일)에서 단계 전환
  - `poseFor` 우선순위(돌봄>졸림>시무룩>평온)
  - 직렬화 라운드트립(serialize→deserialize 동일)
- **수동/Playwright**: 브라우저에서 돌봄→스탯↑·표정 전환, 새로고침 후 상태 유지, 시간 점프 시 오프라인 감소 확인.

## 16. 재활용 가치

`tama-core`의 "상태 1개 → tick → render" 패턴과 rAF 루프·셸 배선은 ①에서 검증된 구조 그대로.
같은 곰(bear-core·poses)을 공유해 표면 간 캐릭터 일관성 유지. 16스프라이트는 ③ Phaser에서도 재활용 가능.
