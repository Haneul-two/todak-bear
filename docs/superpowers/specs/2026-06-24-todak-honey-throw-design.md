# 토닥곰 던지기 (honey-throw) — 설계

- **날짜:** 2026-06-24
- **표면:** 웹 (GitHub Pages, `docs/play/`)
- **단계:** 곰 우주 게임 로드맵 3/3 (① 꿀단지 받기 → ② 다마고치 → ③ **Phaser 던지기**)
- **상태:** 설계 승인됨, 구현 계획 대기
- **목적:** Phaser 게임 엔진 + Matter.js 물리 학습 (①②의 순수 Canvas+rAF 커널을 졸업하고 엔진 도입)

## 1. 한 줄 요약

곰이 **꿀단지를 드래그 새총으로 던져** 장애물을 피하거나 무너뜨리며, 위쪽에 있는 **꿀항아리에 정확히 넣는** 세로형 앵그리버드류 물리 게임. 적 없음, 따뜻·온화한 톤.

## 2. 핵심 결정 (브레인스토밍 확정)

| 항목 | 결정 |
|---|---|
| 무엇 | Phaser로 **새 곰 게임** 신규 제작 (기존 게임 이식 아님) |
| 장르 | 꿀단지 던지기 (앵그리버드 류), 발사체 물리 |
| 목표/승리 | 멀리(위) 있는 **꿀항아리에 꿀단지 넣기** (적 없음·온화) |
| 조작 | **드래그 새총** (곰/새총 당겼다 놓기) + 궤적 미리보기 |
| 레벨 | 고정 **6개** (난이도 상승), 클리어 시 다음 레벨, 별점·진행도 저장. 재미 확인 후 7~8 확장 |
| 물리 | **Matter.js** (Phaser 내장 강체) — 사실적 튕김·구조물 붕괴, 항아리 넣기=센서 판정 |
| 화면 방향 | **세로 360×640** (①② 계열, 한 손 모바일). 던지는 축을 세로로 활용 |

## 3. 기술 스택 & 파일 구조

**표면:** 웹 `docs/play/`, GitHub Pages 정적 서빙, **빌드 없음** (①② 동일).

**Phaser 로딩:** `phaser.min.js`를 **로컬 벤더링** → `docs/play/vendor/phaser.min.js`, `<script src="vendor/phaser.min.js">` 한 줄. CDN 의존 없이 오프라인·재현 가능. (Phaser 3 최신 안정 버전)

**파일 구조** (①②의 "순수로직 ↔ 렌더 분리" 원칙 유지):
```
docs/play/
  honey-throw.html          # 진입점: Phaser 부팅 + 씬 등록
  throw/
    levels.js               # [순수] 레벨 데이터 6개 + validateLevel
    scoring.js              # [순수] 별점·시도횟수·클리어 판정 + 진행도 직렬화
    trajectory.js           # [순수] 드래그→발사벡터·궤적 미리보기 점 계산
    scenes/
      BootScene.js          # 에셋 로드(곰·꿀단지·항아리·통나무), 폴백 처리
      GameScene.js          # Matter 월드·새총·발사·충돌·넣기 판정·상태머신
      UIScene.js            # 레벨·시도수·별점·재시작·다음레벨 (오버레이)
    throw-core.test.js      # [순수 모듈 Node 테스트]
  vendor/phaser.min.js      # 벤더링한 Phaser 3
  assets/poses/             # 기존 토닥곰 스프라이트 재사용
```

**핵심 의도:** 순수 로직(레벨 데이터·점수·궤적 수학)은 Phaser 없이 Node 단위테스트 가능 → ①②와 같은 테스트 가능성. 물리 시뮬레이션·씬은 브라우저 스모크로 검증. 같은 토닥곰 스프라이트 재사용으로 캐릭터 일관성.

## 4. 게임플레이 흐름 & 상태

**한 레벨 흐름:** 레벨 시작 → 조준 → 발사 → 비행/충돌 → 결과 판정 → (단지 남고 실패면 다시 조준 / 성공이면 클리어 연출 → 다음 레벨).

**상태 머신 (GameScene):**
| 상태 | 설명 | 전이 |
|---|---|---|
| `AIMING` | 꿀단지 장전(static 대기), 드래그 대기. 당기는 중 궤적 점선 미리보기 | 놓으면 → `FLYING` |
| `FLYING` | Matter 물리로 꿀단지 비행. 장애물·바닥 충돌 | 항아리 센서 진입(아래방향) → `CLEARED` / 멈춤·이탈 → `SETTLED` |
| `SETTLED` | 꿀단지 정지(넣기 실패). 남은 단지 차감 | 단지 남음 → `AIMING` / 0개 → `FAILED` |
| `CLEARED` | 항아리 성공. 별점 계산·저장·축하(곰 happy+꿀 파티클) | 입력 → 다음 레벨 |
| `FAILED` | 단지 소진 실패. 따뜻한 위로 톤 | 재시작 → `AIMING` |

**투척 자원:** 레벨당 꿀단지 **3개**. 다 쓰기 전 넣으면 클리어.

**별점 (scoring.js, 순수):** ⭐ 1~3. **1발 성공=⭐⭐⭐, 2발=⭐⭐, 3발=⭐.**

**톤(곰 가이드):** 실패해도 "괜찮아, 다시!" 톤. 곰은 화내지 않음(둥글·부드럽·안심, 화난 표정 금지).

**저장(localStorage):** 키 `todak.throw.v1`. 레벨별 최고 별점 + 클리어 여부 + 최고 도달 레벨. 재방문 시 이어서/레벨 선택.

## 5. Matter 물리 & 충돌 (세로형 + 단점 보완)

**화면:** 세로 360×640. **던지는 축을 세로(긴 변 640)로** 활용 → 위로 길게 던지는 시원한 포물선. 곰/새총은 아래쪽, 항아리는 위쪽(살짝 대각).

**세로형 단점 보완:**
1. **세로 비행축** — 640px 높이 전체가 포물선 공간(농구 슛 손맛, 긴 아크).
2. **수직 적층 구조물** — 통나무·상자를 세로로 쌓음. 중력 붕괴는 세로 적층이 더 드라마틱 → Matter 재미 강화.
3. **소폭 카메라 워크** — 발사 후 꿀단지 따라 위로 살짝 팬/줌아웃(연출용; 레벨은 한 화면 높이 기준 설계).
4. **대각 배치 + 좌우 장애물** — 항아리를 위-좌/위-우 대각에 두고 옆에 막는 구조물 → 좁은 가로폭을 정밀 조준 난이도로 전환.

**월드:** 중력 `y: 1`(아래). 바닥·좌우 벽 = 정적 바디(화면 밖 이탈 방지).

**바디 종류:**
| 객체 | 바디 | 물성 |
|---|---|---|
| 꿀단지(발사체) | 동적 원형 | 반발 0.4(살짝 통통), 마찰 있음 |
| 통나무(log) | 동적 사각(길쭉) | 부딪히면 밀리고 굴러감 |
| 상자(box) | 동적 사각(정사각) | 적층, 무너짐 |
| 바위(rock) | 정적 사각 | 고정 벽 |
| 꿀항아리(목표) | **정적 + 센서** | 입구 sensor 바디 → 꿀단지 진입 감지(물리충돌 X) |
| 바닥/벽 | 정적 | 경계 |

**발사 (드래그 새총):**
1. `AIMING`: 곰/새총 지점에 꿀단지 고정(static 대기).
2. 포인터 down→drag: 당김벡터 = (새총원점 − 포인터). `trajectory.js`가 발사 속도 + 궤적 미리보기 점 계산(최대 당김 거리 클램프).
3. 드래그 중 점선 궤적 표시(중력 반영 포물선 샘플 점).
4. 포인터 up: 꿀단지를 동적 바디로 전환 + 속도 부여 → `FLYING`.

**넣기 판정:** 항아리 입구 센서 진입 + (선택) 아래방향 속도 → `collisionStart` → `CLEARED`. 위로 튕겨 나가면 미인정.

**정지 판정(SETTLED):** 속도 임계 이하 일정 시간 유지 OR 화면 밖 이탈 → 실패.

**순수 분리:** `trajectory.js`(당김벡터→속도·궤적점)는 Matter 없이 순수 수학 → Node 테스트. 실제 물리·충돌 이벤트는 GameScene(브라우저 스모크).

**튜닝 리스크(인정):** 반발·마찰·발사 배율·중력은 브라우저에서 손으로 맞춤(스모크 단계에서 "넣을 만한" 난이도로 조정).

## 6. 레벨 데이터 스키마 & 난이도 곡선

**`levels.js` (순수 데이터 + 검증):** 각 레벨은 좌표·배치를 선언하는 객체. Phaser/Matter는 이 데이터를 읽어 바디 생성만 함 → 데이터·엔진 분리.

```js
// 좌표계: 360×640 논리해상도, 원점 좌상단
{
  id: 1,
  bear:    { x: 70, y: 560 },          // 새총/곰 위치(아래)
  pot:     { x: 250, y: 120, r: 36 },  // 항아리 입구 센서 반경
  jars:    3,                           // 지급 꿀단지 수(시도)
  obstacles: [
    { type: 'log',  x: 180, y: 400, w: 90, h: 24, static: false },
    { type: 'rock', x: 250, y: 300, w: 40, h: 40, static: true },
    { type: 'box',  x: 250, y: 240, w: 44, h: 44, static: false },
  ],
  hint: '통나무를 맞춰 길을 트세요'      // 선택: 첫 등장 기믹 안내(ko)
}
```

**바디 타입 3종 고정(YAGNI):** `log`(동적·길쭉·굴림), `box`(동적·정사각·적층), `rock`(정적·고정 벽).

**난이도 곡선 (6레벨, 한 기믹씩):**
| Lv | 새 기믹 | 의도 |
|---|---|---|
| 1 | 장애물 없음, 항아리만 | 던지기·새총 학습 |
| 2 | 고정 바위 1개 | 비껴 던지기 |
| 3 | 통나무(동적) | 맞춰서 밀어내기 |
| 4 | 상자 적층 | 무너뜨리기 |
| 5 | 바위+상자 복합 벽 | 정밀 조준 |
| 6 | 좁은 틈+대각 항아리 | 종합 |

**검증 `validateLevel(lv)` (순수, 테스트):** 필수 필드 존재·좌표 화면 내·jars≥1·항아리와 곰 안 겹침. 깨진 데이터 조기 차단.

**스키마 안정성:** `levels.js`는 버전 없는 정적 데이터(저장 대상 아님). 저장은 진행도만(`todak.throw.v1`), 레벨 수 증가해도 하위호환 영향 없음.

## 7. UI 레이아웃 · 에러 처리

**UI (UIScene 오버레이, 360×640):**
```
┌──────────────────────┐
│ Lv 3   ⭐⭐☆   🍯🍯🍯 │ ← 상단바: 레벨·최고별점·남은 단지
│                       │
│         🍯 (항아리)    │
│       ▦  ▦            │   게임 영역
│  🐻↖ (당기는 중        │
│      점선 궤적)         │
│  [↺ 다시]   [≡ 레벨]   │ ← 하단: 재시작·레벨선택
└──────────────────────┘
```
- 결과 오버레이: 성공=⭐연출+곰 happy+꿀 파티클+[다음], 실패=위로 톤+[다시].
- 조작 힌트: 레벨 1에서만 "곰을 당겼다 놓아 던지세요"(이후 숨김).
- 색·톤: 크림 배경 `#FBF3E4`·따뜻·둥글, ①②와 통일.

**에러 처리:**
| 상황 | 처리 |
|---|---|
| `phaser.min.js` 로드 실패 | "게임을 불러오지 못했어요 🐻" 폴백 메시지 |
| 스프라이트 로드 실패 | BootScene 도형 폴백(원=꿀단지, 사각=장애물)으로 계속 |
| localStorage 차단(시크릿) | try/catch → 인메모리 진행(저장만 생략), 게임 정상 |
| 깨진 저장 데이터 | `deserialize` 버전·타입 검증 실패 → 새 진행도 초기화 (②와 동일) |
| 꿀단지 영원히 안 멈춤 | 속도 임계+타임아웃 → SETTLED 강제, 화면 밖 이탈도 실패 |

## 8. 테스트 전략

**순수 모듈 Node 단위테스트 (`throw-core.test.js`)** — ①②와 동일 방식, TDD 적용:
- `trajectory.js`: 당김벡터→속도 변환, 최대 당김 클램프, 궤적 점 포물선.
- `scoring.js`: 1발=⭐⭐⭐·2발=⭐⭐·3발=⭐, 별점 저장/병합.
- `levels.js`: `validateLevel` 전 레벨 통과, 잘못된 레벨 거부.
- `serialize/deserialize`: 라운드트립, 깨진/구버전 데이터 거부.

**브라우저 스모크 (Playwright)** — 물리·씬: 페이지 로드→레벨1 곰 드래그→발사→항아리 넣기→클리어/별점→다음레벨→localStorage 저장 확인. 콘솔 에러 점검.

**TDD:** 순수 모듈은 test-driven-development(빨강→초록). 물리/씬은 스모크.

## 10. 로드맵 연속성

①(꿀단지 받기)·②(다마고치)의 **"순수 로직 ↔ 렌더 분리"** 원칙을 ③에서도 유지하되, 렌더·물리 계층을 자작 Canvas+rAF에서 **Phaser+Matter**로 졸업. 같은 토닥곰(assets/poses) 공유로 표면 간 캐릭터 일관성. 진입 링크는 랜딩 `docs/index.html`에 `cta_throw`(ko/en)로 추가.
